        @RequestParam:注解用于映射请求中的参数：
        value:对应的请求中URI中的参数名，
                 找对对应的参数后，将这个参数值，注入到@RequestParam注解后面的变量，如下案例是将userid这个参数的值，注给abc这个变量

3、SpringMVC支持RESTFull风格的URL


Http协议本身可以约定前端请求的方式有GET、POST、PUT、DELETE，但是html中的form表单仅仅能发起get或者post请求。

POST:新增（保存、注册）：一般在向服务器端发送一个资源（数据），新书入库，新闻发布，新品上架、用户注册
DELETE：删除，一般要删除服务器端一个资源时（删除表中一条数据），需要送DELETE
PUT：更新：跟新服务器端一个资源（跟新表中的一个数据），需要发起PUT请求
GET：检索（获取、查询）；一般需要从服务器端获取一个资源


常规请求：<a href="userAction/user?userid=rock">删除</a>
RestFull：
<a href ="userAction/user/rock">            删除</a>            DELTET
<a href ="userAction/user/rock">            新增</a>            POST
<a href ="userAction/user/rock">            检索</a>            GET
<a href ="userAction/user/rock">            跟新</a>             PUT

@RequestMapping("userAction/user/rock")
public String doSomeThing(){}


RestFull风格的URL的优势：
Rest不是一个新的技术新的规范，而是对http协议的充分利用后到的的一个URL的风格
①、让这个url具备很强的可读型
②、这个url具有自我描述性
③、很好的资源描述和视图

SpringMVC提供了一个叫做org.springframework.web.filter.HiddenHttpMethodFilter这个拦截器，用于拦截web前端发送过来的请求，拦截后会去识别当前请求的方式是POST|DELETE|PUT|GET。



步骤：
在web.xml配置HiddenHttpMethodFilter拦截所有请求，然后判断请求中的_method的值，然后基于这个值将请求转化不同类型的请求方式
  <!-- 配置SpringMVC的HiddenHttpMethodFilter这个拦截器，可以将web前端的post请求转化为DELETE|PUT|POST|GET等方式 -->
  
  <filter>
     <filter-name>hiddenHttpMethodFilter</filter-name>
     <filter-class>org.springframework.web.filter.HiddenHttpMethodFilter</filter-class>
  </filter>
  <filter-mapping>
          <filter-name>hiddenHttpMethodFilter</filter-name>
          <url-pattern>/*</url-pattern>
  </filter-mapping>
